# Button ‚Äî Complete Guide

A flexible, themeable button component inspired by Material-UI design principles. Use it for actions, navigation, form submissions, and more.

**What makes this Button special?**
- **Three visual styles** ‚Äî Choose between text, outlined, or contained variants
- **Six color palettes** ‚Äî Primary, secondary, success, error, warning, and info colors built-in
- **Three sizes** ‚Äî Small, medium, and large to fit any context
- **Icon support** ‚Äî Add icons before or after the button text
- **Fully themeable** ‚Äî Customize every aspect through the theme provider
- **Accessible** ‚Äî Built on native `<button>` with full keyboard and ARIA support

**Why use this Button?**
- Consistent styling across your entire app
- Easy to customize without writing CSS
- Works great with forms and interactive elements
- Supports all native button features (onClick, type, disabled, etc.)

---

## Quick Start

Here's the simplest way to use a Button:

```tsx
import { Button } from "@karya_app1/rain-js";

function App() {
  return (
    <Button variant="contained" color="primary">
      Click me
    </Button>
  );
}
```

**What's happening here?**
1. Import the Button component from the library
2. Set `variant="contained"` for a solid, filled button
3. Set `color="primary"` to use the primary theme color
4. Add text as children ‚Äî that's what shows on the button

## How it Works ‚Äî Core Concepts

### 1. **Variants control the visual style**
The `variant` prop determines whether your button is filled, outlined, or text-only:
- `contained` ‚Äî Solid background, most prominent
- `outlined` ‚Äî Border with transparent background
- `text` ‚Äî No border or background, minimal style

### 2. **Colors provide semantic meaning**
Six built-in colors help convey the button's purpose:
- `primary` ‚Äî Main actions (default blue)
- `secondary` ‚Äî Secondary actions (default purple)
- `success` ‚Äî Positive actions (default green)
- `error` ‚Äî Destructive actions (default red)
- `warning` ‚Äî Caution actions (default orange)
- `info` ‚Äî Informational actions (default light blue)

### 3. **Sizes adapt to context**
Three sizes for different use cases:
- `small` ‚Äî Compact spaces, tight layouts
- `medium` ‚Äî Default, most common
- `large` ‚Äî Emphasis, call-to-action buttons

### 4. **Theme system for consistency**
Wrap parts of your app with `ButtonThemeProvider` to customize colors, sizes, fonts, and more. All buttons inside automatically inherit the theme.

### 5. **Native button features**
Since it's built on a real `<button>` element, you get everything HTML buttons provide: onClick, onFocus, disabled, type, aria-label, and more.

## API Reference ‚Äî What the Button Returns

When you use `<Button>`, it renders a native HTML `<button>` element with enhanced styling and features.

## ButtonProps ‚Äî All Available Properties

Every prop you can pass to the Button component, explained in detail:

```tsx
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  children?: React.ReactNode;
  variant?: "text" | "outlined" | "contained";
  color?: "primary" | "secondary" | "success" | "error" | "warning" | "info";
  size?: "small" | "medium" | "large";
  disabled?: boolean;
  startIcon?: React.ReactNode;
  endIcon?: React.ReactNode;
  fullWidth?: boolean;
  onClick?: React.MouseEventHandler<HTMLButtonElement>;
  // Plus all native button attributes: type, aria-*, name, value, etc.
}
```

### `children` (optional)
- **Type:** React node (text, elements, components)
- **What it is:** The content displayed inside the button
- **Usage:** Usually text, but can be any React element
- **Examples:**
  ```tsx
  // Simple text
  <Button>Click Here</Button>
  
  // With formatting
  <Button>
    <strong>Bold</strong> action
  </Button>
  
  // Icon only (provide aria-label for accessibility)
  <Button aria-label="Add item" startIcon={<PlusIcon />} />
  ```

### `variant` (optional)
- **Type:** `"text"` | `"outlined"` | `"contained"`
- **What it does:** Controls the visual style of the button
- **Default:** `"text"`
- **When to use each:**
  - **contained** ‚Äî Primary actions, most important button on the page
  - **outlined** ‚Äî Secondary actions, alternatives to the primary action
  - **text** ‚Äî Tertiary actions, minimal emphasis, inline with content
- **Examples:**
  ```tsx
  // Filled button with solid background (most prominent)
  <Button variant="contained" color="primary">Save</Button>
  
  // Outlined button with border (secondary)
  <Button variant="outlined" color="primary">Cancel</Button>
  
  // Text-only button (minimal)
  <Button variant="text" color="primary">Learn More</Button>
  ```
- **Visual comparison:**
  ```tsx
  <div style={{ display: "flex", gap: "12px" }}>
    <Button variant="text">Text</Button>
    <Button variant="outlined">Outlined</Button>
    <Button variant="contained">Contained</Button>
  </div>
  ```

### `color` (optional)
- **Type:** `"primary"` | `"secondary"` | `"success"` | `"error"` | `"warning"` | `"info"`
- **What it does:** Sets the color scheme from your theme
- **Default:** `"primary"`
- **When to use each:**
  - **primary** ‚Äî Main brand actions (Sign Up, Submit, Continue)
  - **secondary** ‚Äî Supporting actions (View Details, Edit)
  - **success** ‚Äî Positive actions (Approve, Confirm, Accept)
  - **error** ‚Äî Destructive actions (Delete, Remove, Cancel Subscription)
  - **warning** ‚Äî Cautionary actions (Proceed with Caution, Override)
  - **info** ‚Äî Informational actions (Learn More, Help, Info)
- **Examples:**
  ```tsx
  // Success button for positive actions
  <Button variant="contained" color="success">Approve</Button>
  
  // Error button for destructive actions
  <Button variant="contained" color="error">Delete Account</Button>
  
  // Warning button for cautious actions
  <Button variant="outlined" color="warning">Proceed Anyway</Button>
  
  // All colors side by side

    <Button variant="contained" color="primary">Primary</Button>
    <Button variant="contained" color="secondary">Secondary</Button>
    <Button variant="contained" color="success">Success</Button>
    <Button variant="contained" color="error">Error</Button>
    <Button variant="contained" color="warning">Warning</Button>
    <Button variant="contained" color="info">Info</Button>
  </div>
  ```

### `size` (optional)
- **Type:** `"small"` | `"medium"` | `"large"`
- **What it does:** Controls the button's padding, font size, and height
- **Default:** `"medium"`
- **When to use each:**
  - **small** ‚Äî Compact UIs, tables, tight spaces, secondary actions
  - **medium** ‚Äî Most common, default for general use
  - **large** ‚Äî Call-to-action buttons, emphasis, hero sections
- **Examples:**
  ```tsx
  // Small button for compact spaces
  <Button variant="contained" size="small">Small</Button>
  
  // Large button for emphasis
  <Button variant="contained" size="large">Get Started Free</Button>
  ```

  - Large: 42px min-height, 0.9375rem font

### `disabled` (optional)
- **Type:** Boolean
- **What it does:** Disables the button (prevents clicks, reduces opacity)
- **Default:** `false`
- **When to use:** 
  - Form is invalid or incomplete
  - Action is not currently available
  - Preventing multiple submissions
  - Loading state
- **Examples:**
  ```tsx

  <Button variant="contained" disabled>Submit</Button>
  
  // Conditional disabling based on form state

  ```
- **Note:** Disabled buttons have 60% opacity and no hover effects

- **Type:** Function that receives a mouse event
- **What it does:** Called when the button is clicked
- **When to use:** For any interactive action
- **Examples:**
  ```tsx

  <Button
  ```

## ButtonThemeProvider ‚Äî Customizing Appearance

The theme provider lets you customize button styling across your entire app or specific sections.

### Basic Usage

```tsx
import { Button, ButtonThemeProvider } from "@karya_app1/rain-js";

function App() {
  return (
    <ButtonThemeProvider
      theme={{
        colors: {
          primary: {
            main: "#4f46e5",
            hover: "#4338ca",
            text: "#4f46e5",
            border: "#4f46e5"
          }
        }
      }}
    >
      <Button variant="contained" color="primary">
        Custom Primary Color
      </Button>
    </ButtonThemeProvider>
  );
}
```

### Theme Structure

```tsx
interface ButtonTheme {
  colors: {
    primary: ButtonColorConfig;
    secondary: ButtonColorConfig;
    success: ButtonColorConfig;
    error: ButtonColorConfig;
    warning: ButtonColorConfig;
    info: ButtonColorConfig;
  };
  sizes: {
    small: ButtonSizeConfig;
    medium: ButtonSizeConfig;
    large: ButtonSizeConfig;
  };
  borderRadius: string;
  fontFamily: string;
  textTransform: string;
  letterSpacing: string;
  fontWeight: number;
}

interface ButtonColorConfig {
  main: string;      // Background for contained variant
  hover: string;     // Background on hover
  text: string;      // Text color for outlined/text variants
  border: string;    // Border color for outlined variant
}

interface ButtonSizeConfig {
  padding: string;
  fontSize: string;
  minHeight: string;
}
```

## Common Examples

### Form Submit Button

```tsx
function LoginForm() {
  const [email, setEmail] = React.useState("");
  const [password, setPassword] = React.useState("");
  const [loading, setLoading] = React.useState(false);
  
  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setLoading(true);
    
    try {
      await fetch("/api/login", {
        method: "POST",
        body: JSON.stringify({ email, password })
      });
      alert("Login successful!");
    } catch (error) {
      alert("Login failed");
    } finally {
      setLoading(false);
    }
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input 
        type="email" 
        value={email}
        onChange={(e) => setEmail(e.target.value)}
```

## Accessibility

The Button component is built with accessibility in mind:

### Keyboard Support
- **Enter/Space** ‚Äî Activates the button when focused
- **Tab** ‚Äî Moves focus to/from the button
- Custom `tabIndex` supported for focus order control

### Screen Readers
- Uses native `<button>` element for proper semantics
- Provide `aria-label` for icon-only buttons
- Disabled state is announced automatically

### Best Practices

```tsx
// ‚úÖ Good: Icon-only button with aria-label
<Button 
  variant="contained"
  aria-label="Add to favorites"
  startIcon={<span>‚≠ê</span>}
/>

// ‚ùå Bad: Icon-only button without aria-label
<Button variant="contained" startIcon={<span>‚≠ê</span>} />

// ‚úÖ Good: Descriptive text
<Button variant="contained" color="error">Delete Account</Button>

// ‚ùå Bad: Ambiguous text
<Button variant="contained">Click Here</Button>

// ‚úÖ Good: Loading state with changed text
<Button disabled={loading}>
  {loading ? "Processing..." : "Submit"}
</Button>

// ‚ùå Bad: No indication of state
<Button disabled={loading}>Submit</Button>
```

### Preventing Multiple Clicks

```tsx

  const [submitting, setSubmitting] = React.useState(false);
```

### Responsive Sizing

```tsx
function ResponsiveButton() {
  const [size, setSize] = React.useState<"small" | "medium" | "large">("medium");
  
  React.useEffect(() => {
    const handleResize = () => {
      if (window.innerWidth < 640) setSize("small");
      else if (window.innerWidth < 1024) setSize("medium");
      else setSize("large");
    };
    
    handleResize();
    window.addEventListener("resize", handleResize);
    return () => window.removeEventListener("resize", handleResize);
  }, []);
  
  return (
    <Button variant="contained" size={size}>
      Responsive Button
    </Button>
  );
}
```

## Troubleshooting

### Button text not showing
- Make sure you're passing `children` prop or text between opening/closing tags
- Check that `children` is not `null` or `undefined`

### Click handler not firing
- Ensure the button is not `disabled`
- Check that `onClick` is passed correctly
- Verify no parent element is preventing event propagation

### Styling not applying
- Check if there's a theme provider overriding your styles
- Inline `style` prop takes precedence over theme
- Use browser DevTools to inspect computed styles

### Icons not aligning
- Ensure icon components have consistent size (16x16px is standard)
- Icons automatically get `currentColor` for fill/stroke
- Check that icon wrapper elements don't add extra spacing

The Button component is a versatile, themeable component that covers all common use cases:

**Key Features:**

- Disabled state

- Provide aria-label for icon-only buttons
- Show loading/disabled states appropriately
- Keep button text concise and action-oriented
- Use consistent theming across your app
